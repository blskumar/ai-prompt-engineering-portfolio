# ai-prompt-engineering-portfolio

A portfolio of structured system prompts for transforming raw requirements into clear Markdown features.

## Purpose

This repository showcases reusable system prompts designed to convert unstructured requirement inputs such as notes, stakeholder requests, emails, and rough feature ideas into clean, organized Markdown output.

## Prompt Files

### 1. system-prompt-1.md
Focuses on converting raw requirements into a structured feature specification with sections such as summary, business goal, functional requirements, inputs, outputs, constraints, and open questions.

### 2. system-prompt-2.md
Focuses on transforming messy stakeholder notes into backlog-ready Markdown using objective, user story, acceptance criteria, scenarios, assumptions, and clarifications needed.

### 3. system-prompt-3.md
Focuses on converting notes, chats, and emails into implementation-ready feature briefs with context, actors, triggers, behaviors, validation rules, exceptions, dependencies, and definition of done.

## Usage

These prompts can be reused as templates for requirement refinement, backlog preparation, and prompt engineering practice.
